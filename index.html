Technical challenges

Technical challenges

A chatbot or feedback system

  • is a centrally managed service,
  • requiring a server side setup.

This poses some challenges.

Technical challenges

  • Tools selection: 4 commercial | 4 free/open source solutions
    • DEXlab's AI tool from bracketlab.ai (applicability on a larger scale?)
    • The Plant's Aparecium Pilot
    • Availability of off-the-shelf tools?
  • Server infrastructure
  • Budget constraints on:
    • the use of commercial tools, services and paid API keys
    • the need for data preparation, fine-tuning, development tasks.

Aparecium (2023)

Accessibility issues

  • System access restrictions: access to institution-specific (possibly non-public) materials, and student profile data might be required.
  • User access restrictions:
    • Institution | student | course only access.
    • Case group access vs. control group.

Ethics and privacy

Privacy considerations

  • AI Inference using cloud based services is a concern:
    • Personal character of student interactions.
    • Potentially copyrighted course materials, guidelines and regulations.
  • Data management plan and ethical clearance will be required in line with GDPR.

Ethical considerations

  • Ecological footprint (consider local models vs. cloud based services)
  • The cognitive offload effect of AI may conflict with the critical thinking and problem-solving skills that we want our students to acquire.
